# 薪资分析可视化应用设计稿

## 1. 项目概述

本项目是一个本地优先的交互式薪资分析可视化应用，面向个人及少量共享用户场景，支持多档案管理、薪资录入与 Excel 导入、月度与年度分析、假设分析、方案对比、本地加密存储，以及 PDF/Excel 导出。

第一阶段按“分析看板优先”的 MVP 思路实施，优先形成可用产品，再在其上扩展更复杂的预测模型与规则包。

## 2. 目标与范围

### 2.1 MVP 目标

- 提供本地网页应用形态，兼容 PC 并适配移动端。
- 支持多档案管理，适合本人及家人/朋友共用。
- 支持手动录入与 Excel 导入两种数据入口。
- 提供分析驾驶舱首页，展示税前、税后、个税、五险一金、福利账户、历史趋势与方案差异。
- 提供假设分析，支持调整五险一金缴纳基数、专项附加扣除、年终奖月份数、调薪幅度等参数。
- 提供方案对比，支持同一档案多方案或不同档案的横向比较。
- 提供本地加密存储与 PDF/Excel 导出。

### 2.2 非目标

- 不做云同步、在线账号体系、多人实时协作。
- 不在第一阶段实现复杂的企业年金收益模型或股票/期权估值模型。
- 不做跨地区极度复杂的社保政策自动联网同步，先以内置规则包与手动配置为主。

## 3. 用户与使用场景

### 3.1 目标用户

- 个人用户：管理自身工资、奖金、个税、社保公积金与年包。
- 少量共享用户：在一台设备上维护多个独立档案。

### 3.2 核心场景

- 查看某档案当前月份和累计年度的薪资构成。
- 导入历史 5 年薪资数据并观察趋势。
- 调整调薪幅度、职级变化和扣除参数，比较多个未来方案。
- 导出脱敏后的 PDF 报告或 Excel 明细。

## 4. 信息架构

主导航固定为以下 5 个一级页面：

- `概览仪表盘`
- `薪资录入`
- `Excel 导入`
- `方案对比`
- `设置`

### 4.1 概览仪表盘

默认首页，采用驾驶舱式布局：

- 顶部工具栏：档案切换、方案切换、日期范围、导出按钮。
- 第一行：KPI 卡片，展示税前、到手、个税、个人五险一金、公司缴纳、年包预测。
- 右侧固定参数栏：假设分析参数，如缴纳基数、专项附加扣除、年终奖月份数、调薪幅度。
- 第二行：月度薪资构成图表，支持税前/税后/公司视角切换。
- 第二行右侧：五险一金分类统计、福利账户余额。
- 第三行：拖拽式时间轴与历史趋势。
- 第三行右侧：对比摘要与异常提醒。
- 底部：可展开的明细表格，按月份与薪资项下钻。

### 4.2 薪资录入

负责创建或编辑月度记录，包含：

- 基本工资、绩效、奖金、补贴、股票/期权。
- 个人与公司五险一金参数。
- 专项附加扣除。
- 福利账户余额。

保存后触发计算引擎重算，并刷新首页摘要。

### 4.3 Excel 导入

负责批量数据导入，流程固定为：

- 下载模板。
- 上传文件。
- 字段映射。
- 行级校验。
- 导入预览。
- 确认写入。

### 4.4 方案对比

负责展示多个方案的差异，支持：

- 基准方案与派生方案对比。
- 不同档案之间的横向对比。
- KPI、图表和年包差异并排展示。

### 4.5 设置

负责：

- 主口令与解锁设置。
- 默认缴纳规则与地区规则选择。
- 税率和规则包版本展示。
- 导出偏好与脱敏选项。

## 5. 核心交互流

### 5.1 录入流

`概览仪表盘 -> 新建记录/编辑月份 -> 保存 -> 即时重算 -> 更新图表与年包测算`

### 5.2 导入流

`Excel 导入 -> 字段映射 -> 预览校验 -> 批量入库 -> 返回首页查看趋势`

### 5.3 对比流

`首页选择多个方案 -> 进入方案对比页 -> 并排展示指标、图表、年包差异`

### 5.4 假设分析流

`首页右侧参数调整 -> 节流触发重算 -> 更新当前方案结果 -> 不覆盖原始记录`

## 6. 数据模型

### 6.1 Profile

档案实体，保存用户级基础配置：

- 档案名称、昵称。
- 城市或地区。
- 默认缴纳规则。
- 默认专项附加扣除模板。
- 解锁与显示偏好。

### 6.2 PayrollRecord

月度薪资记录：

- 月份。
- 基本工资。
- 绩效。
- 奖金。
- 补贴。
- 股票/期权。
- 个税。
- 税后到手。
- 年度归属信息。

### 6.3 SocialFund

社保公积金记录：

- 养老保险。
- 医疗保险。
- 失业保险。
- 工伤保险。
- 生育保险。
- 住房公积金。

每项均区分个人缴纳与公司缴纳。

### 6.4 BenefitAccount

福利账户记录：

- 企业年金。
- 补充公积金。
- 商业保险。
- 其他福利账户余额。

### 6.5 DeductionProfile

专项附加扣除配置：

- 子女教育。
- 继续教育。
- 住房贷款利息。
- 住房租金。
- 赡养老人。
- 其他可扩展项。

### 6.6 Scenario

方案实体，用于模拟与对比：

- 方案名称。
- 基准记录引用。
- 调薪幅度。
- 职级变化。
- 年终奖月份数。
- 缴纳基数假设。
- 专项扣除假设。

派生方案只存差异参数与结果，不复制整份历史数据。

### 6.7 ImportBatch

导入任务实体：

- 原始文件元信息。
- 字段映射关系。
- 逐行校验结果。
- 导入日志。
- 错误汇总。

## 7. 系统分层设计

### 7.1 UI 层

负责：

- 图表展示。
- 表单录入。
- 时间轴交互。
- 方案切换与对比。
- 导入导出入口。

### 7.2 应用层

负责组织业务流程：

- 档案服务。
- 方案服务。
- 导入服务。
- 导出服务。
- 本地存储服务。

### 7.3 计算层

负责纯计算逻辑：

- 个税计算。
- 五险一金计算。
- 年终奖计算。
- 年包汇总。
- 调薪预测与职级模拟。

所有计算函数保持纯函数风格，便于单元测试和回归验证。

### 7.4 规则层

负责维护可版本化规则：

- 个税税率表。
- 起征点。
- 专项附加扣除规则。
- 社保公积金比例与上下限。
- 年终奖计税规则。

### 7.5 存储层

负责本地持久化：

- `IndexedDB` 保存结构化数据。
- 敏感字段加密后再写入存储。
- 保存原始输入、计算结果快照和规则版本号。

## 8. 计算与准确性策略

### 8.1 精度策略

- 所有金额计算统一使用高精度小数库。
- 禁止直接使用原生浮点完成关键财务计算。
- 金额展示与内部计算精度分离，内部精度高于界面显示精度。

### 8.2 结果快照

每次保存后同时保留：

- 原始输入数据。
- 计算结果。
- 规则包版本。
- 计算时间戳。

这样可在规则变更后做重算，并追踪历史口径。

### 8.3 年包定义

年包总收益默认包含：

- 基本工资。
- 绩效。
- 年终奖。
- 补贴。
- 销售或项目类奖金。
- 股票/期权。
- 福利账户类收益。

允许用户按方案设置是否纳入某些项目。

## 9. 本地安全与隐私保护

### 9.1 存储策略

- 使用 `IndexedDB` 作为本地主存储。
- 使用 `Web Crypto` 对敏感字段进行加密。
- 敏感信息包括身份信息、工资明细、专项附加扣除、备注等。

### 9.2 解锁方式

- 应用启动时输入主口令解锁本地数据。
- 多档案共享一台设备时，通过档案级隔离保证数据边界。

### 9.3 导出保护

- 导出前二次确认。
- 支持脱敏导出，如隐藏真实姓名、身份证号、敏感备注。

### 9.4 隐私边界

- MVP 不接入云端。
- 默认仅本地运行。
- 不采集在线遥测数据。

## 10. Excel 导入与数据导出

### 10.1 Excel 导入

导入必须经过以下校验：

- 必填字段校验。
- 数值范围校验。
- 日期和月份格式校验。
- 同月记录冲突校验。
- 字段映射完整性校验。

未通过校验的数据不得直接入库。

### 10.2 Excel 导出

支持：

- 月度明细导出。
- 年度汇总导出。
- 方案对比导出。

导出结果应可再次导入，保证模板兼容。

### 10.3 PDF 导出

支持：

- 分析报告导出。
- 当前方案摘要导出。
- 对比报告导出。

PDF 以清晰打印布局为优先，而非完全复刻交互界面。

## 11. 技术选型

- 前端框架：`React + TypeScript + Vite`
- 图表库：`ECharts`
- 本地数据库：`IndexedDB`
- 数据库封装：`Dexie`
- 金额计算：`decimal.js`
- Excel：`SheetJS`
- PDF：浏览器打印样式 + PDF 生成方案
- 样式：`Tailwind CSS` 或同级轻量方案

选择依据：

- 适合本地网页应用。
- 易于快速搭建响应式仪表盘。
- 便于高精度计算与复杂交互组合。

## 12. 性能设计

### 12.1 性能目标

- 图表渲染响应时间不超过 500ms。
- 支持 5 年历史数据加载与切换不卡顿。
- 假设分析参数调整后在可感知范围内快速重算。

### 12.2 优化策略

- 将原始数据与聚合结果分离。
- 首页优先读取预计算摘要。
- 时间轴和趋势图按月份聚合。
- 明细表采用懒加载或分段展开。
- 假设分析使用节流重算。
- 对比方案只保留差异参数与结果。

## 13. 响应式设计

### 13.1 PC

- 以驾驶舱分析布局为主。
- 假设分析参数栏固定在右侧。
- 适合图表并排和表格展开。

### 13.2 移动端

- 首页采用纵向堆叠。
- KPI 卡片改为两列或单列。
- 图表支持横向滚动。
- 明细由表格切换为卡片形式。

## 14. 测试与验收

### 14.1 准确性测试

覆盖以下场景：

- 个税阶梯计算。
- 专项附加扣除。
- 年终奖单独计税。
- 调薪幅度变化。
- 职级变化。
- 不同缴纳基数。
- 奖金和补贴纳入口径变化。

目标误差率小于 0.01%。

### 14.2 性能测试

覆盖以下场景：

- 5 年历史数据加载。
- 档案切换。
- 方案切换。
- 时间轴拖拽。
- 图表联动刷新。

### 14.3 集成测试

覆盖以下流程：

- 录入并保存。
- Excel 导入并校验。
- 假设分析重算。
- 方案对比。
- PDF/Excel 导出。

### 14.4 端到端验收标准

- 薪资计算误差率小于 0.01%。
- 图表渲染不超过 500ms。
- 5 年数据可流畅浏览。
- 通过复杂个税和年终奖计税验证。

## 15. 分阶段实施建议

### 15.1 第一阶段

- 多档案与解锁。
- 手动录入。
- Excel 导入。
- 概览仪表盘。
- 历史趋势。
- 假设分析。
- 方案对比。
- PDF/Excel 导出。

### 15.2 第二阶段

- 更复杂的预测模型。
- 股票/期权估值。
- 更细粒度地区规则包。
- 本地备份与恢复。

## 16. 结论

本设计采用“本地网页应用 + 分析驾驶舱 + 多档案 + 高精度计算引擎”的路线，先满足真实可用的薪资分析与方案对比需求，再逐步扩展更复杂的理财与预测能力。MVP 重点是准确、可视、易录入、可对比、重隐私。
